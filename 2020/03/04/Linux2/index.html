从终端开始说起

概述

Linux系统和Windows系统最大的不同之处是前者大部分操作都是在shell终端上操作，后者大部分都是在图形界面的操作。在linux里面可谓是终端的世界。上手linux操作系统的第一步就是从终端开始，终端的世界太广阔了，博主也只是简单了解了，但是几乎没有一个人可以说自己把终端完全学会了，因为终端也是不断更新的，命令也在不断增加，总有新的东西等待我们去发掘。

为什么要终端

或许有的人会问，为什么要有终端这东西，为什么Linux操作系统里面会是终端的世界，用图形界面不香吗？现在都什么时代了，还用终端这种老东西，你看看图形界面多好看，多方便。用终端不会是为了装技术高超啊？

  • 为什么要终端？我只能说，终端这东西几十年前就已经发明出来了，当时图形界面还没有出来，终端界面用的过程其实也很方便，沿用至今。但是在技术不断更新迭代的过程中，为什么终端没有成为历史的尘埃呢？的确，学习图形界面的使用会比终端的使用要那么有趣一点，然后也简便易学一点，但是真正学会了终端操作，其实可以做到比图形界面的操作更加方便，更加快。而且有的操作是必须在终端上进行的，图形界面无法做到或者说有的操作在终端上进行更加方便。比如说查看系统是否有Java环境，我们不需要一一去查看那个目录文件里面是否有JDK，只有在终端中输入Java就行了，很快。而且在Linux里面，很多安装下载任务也是在终端上可以运行的，看起来很舒服，几行命令打天下。
  • 很多时候终端命令更加简洁，节省时间

所有，好好学会如何使用Linux操作系统里面的终端吧！！！学完就离坐着奔驰，骑着宝马，迎娶白富美的目标更进一步了(嘿嘿，开玩笑，活跃下气氛)

两种终端

1.真正的终端：真正的终端就是真的是所有操作只能够在终端上进行，没有任何图形操作，系统所有操作都是在这个真正的终端进行的，此时整个系统就是一个黑黑的屏幕，只有字符。如何进入呢？

图形终端的两种模式

1.普通用户模式：我们一进入图形终端的时候，看到的就是普通用户模式，它能够干的事情有限，一般是出现：liujunchi@ubuntu:~$.

2.超级用户模式(root模式)：这个就是相当于管理员模式，基本上什么操作都可以，拥有更多的权限，输入命令：sudo su,然后在输入你的登录密码就可以进入超级管理员模式了，出现的是这个：root@ubuntu:/home/liujunchi#

当你想要退出超级管理模式的时候，可以按快捷键：ctrl+D，就可以退出到普通用户模式了。

基本的文件操作命令

命令参数

我们有很多操作命令都需要使用到命令参数，告诉这个指令他应该干什么，就比如说查看文件的命令，输入参数会告诉这个命令查看文件的什么，比如说字数，行数，字节等等。格式为：command parameters.前者为命令，后者为参数。

参数里可以包含多个参数，由空格隔开，也可以包含数字、字母等等。可以说，参数没有固定的格式，但是一般来说还是遵循一定规范的。

  • 短参数：这个一般都是一个短横线+一个字母，比如说：command -p,也可以一次性加很多个短参数：command -p -a -T -c,当然上述的多个短参数可以合并在一起：command -paTc.
  • 长参数：长参数是两根短横线+一个比较长的单词参数：command --parameter,当然也可以接多个长参数，就像上面的短参数一样，但是这个不能合并。短参数和长参数可以一起使用：command -paTc --parameter1 --parameter2.

文件的组织

1.Linux系统的目录结构：普通用户的根目录为~目录，root用户的根目录为home目录，在Linux系统(虚拟机)里面，只有一个根目录，不像Windows系统有几个根目录，比如说C盘，D盘等等。

以下是Linux的目录结构：

  • bin：英语 binary 的缩写，表示“二进制文件”（我们知道可执行文件是二进制的）。包含了会被所有用户使用的可执行程序；
  • boot：英语 boot 表示“启动”，包含与 Linux 启动密切相关的文件；
  • dev：英语 device 的缩写，表示“设备”，包含外设。它里面的子目录，每一个对应一个外设。比如代表我们的光盘驱动器的文件就会出现在这个目录下面；
  • etc：etc 有点不能顾名思义了。因为 etc 是法语 et cetera 的缩写，翻成英语就是“and so on”，表示“…等等”，包含系统的配置文件。至于为什么在 /etc 下面存放配置文件， 按照原始的 Unix 说法（Linux 文件结构参考 Unix 的教学实现 MINIX），这下面放的都是一堆零零碎碎的东西， 就叫 etc 好了。哈哈 ，这其实是个历史遗留；
  • home：英语 home 表示“家”，用户的私人目录。之前我们提过一些，在这个目录中，我们放置私人的文件，有点类似 Windows 中的 Documents 这个文件夹，也叫“我的文档”。Linux 中的每个用户（除了大管家用户，也就是超级用户 root 外。root 因为太厉害，拥有所有权限，所以比较“任性”，跟普通用户不住在一起）都在 home 目录下有自己的一个私人目录。比如我的用户名是 oscar，那么我的私人目录就是 /home/oscar；如果另一个用户叫 john，那么他的私人目录就是 /home/john；
  • lib：英语 library 的缩写，表示“库”，包含被程序所调用的库文件。例如 .so 结尾的文件，在 Windows 下这样的库文件是以 .dll 结尾的；
  • media：英语 media 表示“媒体”。当一个可移动的外设（比如 USB 盘、SD 卡、DVD、光盘等等）插入电脑时，Linux 就可以让我们通过 media 的子目录来访问这些外设中的内容。
  • mnt：英语 mount 的缩写，表示“挂载”。有点类似 media，但一般用于临时挂载一些装置；
  • opt：英语 optional application software package 的缩写，表示“可选的应用软件包”，用于安装多数第三方软件和插件；
  • root：英语“根”的意思。超级用户 root 的家目录/主目录。一般用户的家目录是位于 /home 下，不过 root 用户是个例外。之前的课程我们也提到过，root 是整个系统的超级用户，拥有一切权限，初学者请慎用此用户模式；
  • sbin：英语 system binary 的缩写，表示“系统二进制文件”。比起 bin 目录多了一个前缀 system，所以包含的是系统级的重要可执行程序；
  • srv：英语 service的缩写，表示“服务”。包含一些网络服务启动之后所需要取用的数据；
  • tmp：英语 temporary 的缩写，表示“临时的”。普通用户和程序存放临时文件的地方；
  • usr：英语 Unix Software Resource 的缩写，表示“Unix 操作系统软件资源”（也是个历史遗留的命名）。这个目录是最庞大的目录之一。有点类似 Windows 中的 C:\Windows 和 C:\Program Files 这两个文件夹的集合。在这里面安装了大部分用户要调用的程序；
  • var：英语 variable 的缩写，表示“动态的，可变的”。通常包含程序的数据，比如一些 log（日志）文件，记录电脑中发生了什么事。

3.head和tail命令查看文件内容的开头和结尾。

  • head命令显示文件的开头，默认情况下，是显示文件内容的前10行，比如说输入：head test.txt，就会显示出test文件内容的前10行。当然如果说我们指定要看多少行可以加入n参数：head -n 5 test.txt 就是显示test文件内容的前5行。

  • tail命令显示文件的结尾，默认情况下显示文件内容的后10行，当然我们也可以像上面一样指定显示的行数，具体形式和上面一样，博主就不多说了。当然，tail还不止这点本事，它还可以实时追踪文件的更新，输入tail -f test.txt 这个时候就会跟踪更新内容，如果更新，后面还会继续出现内容。当然还可以加入s参数设置追踪更新内容的时间，比如说输入tail -f -s 4 test.txt 就是每隔4秒看一下内容的更新。

4.touch命令创建文件：后面加上文件名即可以创建好对应的文件，文件种类任意，比如说其他文件(不带后缀)或者txt文件(后缀为.txt)：touch test和touch test.txt。当然touch可以同时创建多个文件，它后面可以接多个文件名从而创建这些文件：touch file1 file2。这个就创建了file1和file2文件。如果说文件名带有空格，那么就可以用双引号括起来：touch “test file”

5.mkdir命令创建目录：这个用法和上面类似，可以创建一个目录，也可以同时创建多个目录，不过不同的是它可以递归地创建目录(把这个目录以及它的子目录都创建好)：mkdir -p 1/2/3 加上p参数即可，这个就创建了1目录，以及它的子目录2，和子子目录3。有意思吧。

复制删除移动链接文件

1.cp命令实现文件的复制：cp不是我们所说的情侣CP，而是copy的缩写，因此用来复制。

  • 把一个文件复制到另外一个文件：cp file1 file2 这个就会把file1文件复制到file2中，前提是file1存在，如果说file2文件不存在，那么就会创建file2这个文件，且file2文件内容是file1文件的复制。当然，如果file2存在，file1的内容就会复制到file2中，覆盖原来内容。
  • 把一个文件复制到目录中：cp 文件名 目录名。如果说这个目录本身就是当前目录的子目录，可以直接输入目录的名字，如果说这个目录不在当前目录中，那么就需要加上这个目录的路径了。如果说你复制到目录中以后，想把文件名改为想要的，直接目录名后接重命名的名字即可，示例如下：
  • 拷贝一个目录：拷贝目录的时候，目录中所有的文件或者子目录都会被拷贝，这个命令需要加上r参数，进行递归的拷贝，因为目录当中可能还有子目录：cp -r dir1 dir2
  • 使用通配符’‘：通配符是比较常用的正则表达式，可以代替任何字符串，比如说cp *.txt folder,这个就可以把当前目录所有的.txt文件复制到folder目录中，因为*可以代替任何字符串，比如说1.txt,2.txt,3.txt都可以被.txt代替。

2.移动文件或者目录： 移动不像复制，只是单纯地移动文件或者目录，不会生成副本。

  • 移动文件或者目录：mv file1 dir1，假设file1是文件，dir1是当前的子目录，那么file1就可以通过这个命令直接移动到dir1中，此时不会生成副本，是自己亲自出手，因此当前目录不再有file1文件，此时它身在dir1.移动目录也同上，第一个为被移动的，第二个为去哪儿。
  • 重命名：如果说你看某个文件的名字不爽，那么你也可以通过这个命令改名字：mv 当前名字 想要重命名的名字，假设输入：mv fil1 file2，那么file1的名字就会变成file2，但是前提是file2这个名字在这个目录中本身是不存在的，否则就会被移动到这个文件或者目录中，不会重命名。

3.rm删除文件或者目录：rm是remove的缩写，用来删除。

  • 单独使用，不加参数：rm file1，这样就可以删除file1的文件(当前目录中)，但是这样只能删除文件，要删除目录的话，还得加上参数，这个问题暂时留着。
  • 加上i参数询问删除：rm -i file1，执行这个命令，终端会问你是否确认删除，叫你输入Y/N，你想删除的话就输入Y，不想的话就输入N，当然这个命令也是为了保险，放在你发脾气随便删文件，一询问或许就会消消气。
  • 加上f参数暴力删除：这个命令就很危险了，你一输入就无法挽回你的错误(当然还是有办法的，但是会特别麻烦，博主也不懂)，尽量少用这个命令吧，你输入：rm -f file1，真的就直接暴力删除了file1文件了，你无法挽回了。
  • 加上r参数递归删除目录：诶，刚刚不是说要删除目录嘛，这个参数就OK了，比如说：rm -r dir1，这样就可以把dir1目录当中所有的文件(包括这个目录和子目录)都会删除。但是呢，你也可以加上f参数：rm -rf 啥啥啥，这个就会强制性删除了，不过你千万别再加上通配符’*’，不然就是惊天动地，你的东西可能si光光了。

4.ln链接文件: ln 是 link 的缩写，在英语中表示“链接”。所以 ln 命令用于在文件之间创建链接。

  • 创建硬链接：不加任何参数，直接用ln就可以创建文件之间的硬链接了：ln file1 file2 ，这个就可以在file1和file2之间创建硬链接，如果没有file2这个文件，那么就会自动创建，此时file1和file2指向同一个文件内容，也就是说他们共享文件内容，file1文件内容改变了，file2的文件内容也会改变，而且和file1一模一样。因此，file1和file2的Inode是一样的(Inode大家可以百度一下，也可以看看我的博客《玩转操作系统》 里面讲的比较清楚) 。
  • 加s参数创建软链接：创建软链接以后，两个文件的内容也是一样的，但是指向不一样：ln -s file1 file3，就是创建了file1和file3之间的软链接，此时file1指向它的文件内容，file3指向file1，而不是文件内容，但是二者内容还是一样的，毕竟file3指向file1嘛。如果删除了file3，当然不会对file1有影响，但是删除file1以后，file3就会出大事，其就会损坏，因为没了头头。二者inode也是不一样的。

探秘用户和群组

1.adduser命令添加新用户：只不过着命令需要权限，普通用户下可没这个权力，因此需要进入root超级大管家模式才可以执行，执行以后会叫我们输入用户密码，输入完以后，其它的文件名啥的就直接enter不用管，最后确认信息正确就OK了，查看家目录可以看到两个用户文件夹。

2.passwd改用户密码：如果你想改密码，那么就可以通过这个命令实现，后面输入你想要改的用户的名字，如passwd liujunchi。

3.deluser删除用户：这个要在root模式下执行，如果说你不想要这个用户了，可以用这个来删除：deluser thomas，但是这个仅仅只是删除了用户，没有删除在home目录下的用户目录thomas，如果你想一波删除，可以：deluser --remove-home thomas.

4.addgroup创建群组：对于群组我们之前也说过了，利用ls -l查看文件的时候可以看到：

我们输入addgroup friends以后就可以创建名字为friends的群组了，但是此时这个群组是空的，因此还需要添加用户。

5.usermod修改用户：这个可以修改用户名和用户所在的群组等等。

usermod 命令参数比较多，可以实现不同的功能。不过我们暂时只需要记得它的两个参数：

  • -l：对用户重命名，但是 /home 目录中的用户家目录名不会改变，需要手动修改；
  • -g：修改用户所在群组。此用户的家目录里的所有文件的所在群组会相应改变。

usermod -g friends thomas：把用户thomas所在群组改为friends

usermod -g thomas thomas1：把用户thomas的名字改为thomas1

6.delgroup命令删除群组：这个可以删除某个群组，比如说delgroup friends就是删除friends群组，输入groups命令可以查看群组。比如说groups thomas：就会出现：thomas：thomas，就是查看用户thomas所在的群组，发现其在群组thomas中。

文件权限管理

1.chown修改文件的所有者：chown 是 change 和 owner 的缩写，change 是英语“改变”的意思，owner 是英语“所有者”的意思，因此可以用来修改所有者。这个需要在root模式下进行，一般后面是先接用户名，后接文件名，就可以把文件的所有者改为这个用户了。通过ls -l 某个文件可以查看这个文件的所有者。

当然，你需要快速让一个用户占领一个目录当中的所有文件或者子目录，可以加上R参数：chown -R liujunhi:liujunchi /home/thomas 这样就可以把/home/thomas目录当中所有文件和子目录递归地占为己有，把其所有者和群组都变成liujunchi。

2.chgrp改变文件的群组：chgrp 是 change 和 group 的缩写，change 是英语“改变”的意思，group 是英语“群组”的意思。chgrp 命令用于改变文件的群组。用法和上面一样。

3.chmod修改文件访问权限：这个可以修改文件的读写执行的权限。

我们先输入ls -l查看各个文件的详细信息：

第一列哪个rw啥啥啥就是这个文件或者目录的权限，我们来看看每个字母的意思：

  • d：英语 directory 的缩写，表示“目录”。就是说这是一个目录；
  • l：英语 link 的缩写，表示“链接”。就是说这是一个链接；
  • r：英语 read 的缩写，表示“读”。就是说可以读这个文件；
  • w：英语 write 的缩写，表示“写”。就是说可以写这个文件，也就是可以修改；
  • x：英语 execute 的缩写，表示“执行，运行”。就是说可以运行这个文件。

那为什么会有重复的r啥啥，然后还有很多杠，博主来解释一下吧：

除了第一个符号表示的是文件的类型( d，表示目录；如果是 l，则是链接；还有其它字母，我们暂时不深究；如果是短横 - ，那么是普通文件)，其它的符号被’-‘分为三组：

  • 第一组 rwx 表示文件的所有者对于此文件的访问权限；
  • 第二组 rwx 表示文件所属群组的其他用户对于此文件的访问权限；
  • 第三组 rwx 表示除前两组之外的其他用户对于此文件的访问权限。

来看看一个例子，就拿上面的第一行信息为例吧(从左到右)：

-rw-r–r– 1 liujunchi liujunchi 0 Mar 4 02:24 1

  • 第一个符号’-‘代表的是1这个文件是普通文件
  • rw-表示这个文件在所有者当中是可读可写的
  • r–表示这个文件在所属群组的其它用户里是可读的
  • 之后的r–是表示这个文件在其它用户中是可读的

真正的重点来了！！！来看看如何修改文件的权限吧：

  • 其实Linux系统为每一种权限都分配了数字：
权限 数字
r 4
w 2
x 1

一般根据数字的值来分配文件权限，一下是各种可能的组合：

权限组合 数字 计算
– 0(0就是什么权限都没有) 0+0+0
r- 4 4+0+0
-w- 2 0+2+0
-x 1 0+0+1
rw- 6 4+2+0
-wx 3 0+2+1
r-x 5 4+0+1
rwx 7 4+2+1
  • 通过数字分配权限，对用户+所属群组其它用户+对其它用户这三组权限，分配的时候，需要把数字连在一起，比如说：chmod 765 1 就是分别给这个文件在所属用户分配可读可写可执行的权限，所属群组其它用户分配可读可写的权限，其它用户分配可读可执行的权限。

  • 通过字母分配：我们还可以通过不同的字母和符号来进行分配，来看看不同字母的表达意思：

    • u：user 的缩写，是英语“用户”的意思。表示所有者；

    • g：group 的缩写，是英语“群组”的意思。表示群组用户；

    • o：other 的缩写，是英语“其他”的意思。表示其他用户；

    • a：all 的缩写，是英语“所有”的意思。表示所有用户。

      这些字母还需要配合一些符号使用：

      • +：加号，表示添加权限；
      • -：减号，表示去除权限；
      • =：等号，表示分配权限。

比如说，输入nano -miA file1就可以激活这三个功能了(在进入file1中编辑时)

当然大家还会好奇，手册里面看不懂，每部分内容又是什么，如何去看，大家看看下面的关于手册内容的总结：

  • NAME ：英语“名字”的意思。手册页对应的命令或函数名字，后接简单描述；
  • SYNOPSIS ：英语“概要，梗概，大意，摘要”的意思。使用此命令的所有方法。下面我们会详述这个区域，因为此区域的内容极为关键；
  • DESCRIPTION ：英语“描述”的意思。命令的更深入的描述，这个区域也会包括所有参数及其用法。一般来说这个区域是文字最多的；
  • AUTHOR ：英语“作者”的意思。命令的作者，有时候不止一个作者；
  • REPORTING BUGS ：英语“报告故障”的意思。如果使用时遇到 bug（程序的缺陷），有汇报问题的联系方式，通常是邮件；
  • COPYRIGHT ：英语“版权”的意思。版权，也就是用了什么许可证（Licence），大多数命令都是在 GPL 许可证之下的开源代码；
  • SEE ALSO ：英语“另见”的意思。与此命令有关的其它命令，也就是扩展阅读。

  • … ：省略号表示可以有多个此类内容。在 [OPTION] 和 DIRECTORY 后都有省略号，分别表示可以有多个选项和多个目录名或者文件名，可以同时输入多个。

查找文件

1.locate命令快速查找文件： 这个命令后面直接接文件名或者目录名即可，输入完以后，终端会打印出所有含有该文件名的文件的路径，举例如下：

但是这个只能够用于快速查找，而且这个查找是在文件数据库里面查找，文件数据库之外的文件查找不到，比如说我现在创建了一个file1的文件，但是我马上查找的时候，发现查找不到这个文件，那是因为这个文件刚刚创建，没有进入到文件数据库里面，因此无法查找，这也是这个命令的一个缺点，不过还是够用的：

2.find命令深入查找：这个命令就不像上面那个，不是在文件数据库里面查找，而是在整个硬盘里面查找，如果硬盘容量比较大的话，那么就会遍历一段时间了。

  • 根据名字查找(需要name参数)：不管什么参数，都可以指定在某个目录下查找，如果说没有指定，那么真的就是全局硬盘查找了。输入：find -name "文件名字" 这个就会在当前目录以及子目录中查看某个文件了，输入find directory -name "文件名" 这个就可以指定在directory目录里面查找这个文件名对应的文件路径了。
  • 根据文件大小查找(需要size参数)：也可以指定目录，需要接size参数，后面还得加上文件大小,比如说：find -size 10M 这个就会在当前目录以及子目录中查找大小为10M的文件了，再比如：find 1/2 -size +10M，这个就可以在1/2目录下查找文件大小大于10M的文件了，-10M就是小于10M了。
  • 根据最近访问时间查找(需要atime参数)：比如说输入find -atime 10 就是表示查找访问时间在10天的文件，如果-10就是十天内，+10就是10天前。
  • 可以指定查找目录或者文件(需要type)：
    • -type d ：只查找目录类型。d 是 directory 的首字母，表示“目录”；
    • -type f ：只查找文件类型。f 是 file 的首字母，表示“文件”。

比如说输入find /home/liujunchi -name "new_file" -type d 可以查找new_file的目录了，当然，如果出现了Permission denied，可以在命令前面加上sudo，用管理页模式查找。
